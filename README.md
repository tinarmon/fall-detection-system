
# 🚶‍♂️ การพัฒนาระบบตรวจจับการเสียการทรงตัวของบุคคลโดยใช้เทคโนโลยี MediaPipe
**(Development of a Loss of Balance Detection System for Individuals using MediaPipe Technology)**

โปรเจกต์นี้เป็นส่วนหนึ่งของโครงร่างปริญญานิพนธ์ มหาวิทยาลัยเทคโนโลยีราชมงคลล้านนา ตาก มีวัตถุประสงค์เพื่อพัฒนาระบบเฝ้าระวังและแจ้งเตือน "สภาวะก่อนการล้ม" (Pre-fall / Loss of Balance) ล่วงหน้า โดยอาศัยการประมวลผลวิดีโอแบบเรียลไทม์ โครงการนี้เปลี่ยนกระบวนทัศน์จากการตรวจจับเมื่อเกิดแรงกระแทกแล้ว (Post-fall) เป็นการตรวจจับความผิดปกติของท่าทางผ่านโครงร่างมนุษย์ (Skeleton-based) เพื่อรักษาความเป็นส่วนตัวของผู้ใช้งาน

---

## 🏗️ โครงสร้างสถาปัตยกรรมของระบบ (System Architecture)
ระบบถูกออกแบบการทำงานในรูปแบบ **Data Pipeline** ที่ประมวลผลบน Edge Computing โดยแบ่งออกเป็น 5 เลเยอร์หลัก:

1. **Input Video Stream (เลเยอร์รับภาพ):** รับสัญญาณภาพวิดีโอจากกล้องเว็บแคม (ความละเอียดแนะนำ 1080x720p) ผ่านไลบรารี OpenCV เพื่อนำมาประมวลผลเฟรมต่อเฟรม
2. **Pose Extraction (เลเยอร์สกัดพิกัดโครงร่าง):** ใช้เฟรมเวิร์ก MediaPipe Pose (BlazePose) ในการสแกนและแปลงภาพบุคคลให้กลายเป็นพิกัดจุดร่างกาย (Landmarks) จำนวน 33 จุด ในรูปแบบ 3 มิติ (x, y, z)
3. **Feature Engineering (เลเยอร์ประมวลผลฟีเจอร์):** เพื่อลดการใช้ทรัพยากร ระบบจะคัดกรองเฉพาะพิกัดข้อต่อที่ส่งผลต่อสมดุลร่างกาย 6 จุด ได้แก่ ไหล่ (11, 12), สะโพก (23, 24), และเข่า (25, 26) จากนั้นนำมาคำนวณหาความสัมพันธ์เชิงมุม (raw_coord_angle) แยกเป็นซีกซ้ายและซีกขวา
4. **Classification (เลเยอร์วิเคราะห์และจำแนก):** ป้อนข้อมูลมุมที่เปลี่ยนแปลงอย่างต่อเนื่อง (Time-series Data) เข้าสู่โมเดลปัญญาประดิษฐ์โครงข่ายประสาทเทียมแบบ **GRU (Gated Recurrent Unit)** ซึ่งมีจุดเด่นด้านการประมวลผลข้อมูลลำดับเวลาที่รวดเร็วและใช้ทรัพยากรต่ำ เพื่อตัดสินว่าท่าทางปัจจุบันอยู่ในสภาวะสมดุลหรือไม่
5. **Real-time Alert (เลเยอร์แจ้งเตือน):** เมื่อโมเดล GRU ประเมินพบพฤติกรรมการเคลื่อนไหวที่เข้าข่ายเสียการทรงตัว ระบบจะแสดงแถบสีแดงและข้อความ **"WARNING: LOSS OF BALANCE!"** ทันทีก่อนเกิดการล้มกระแทกพื้น

```text

├── train_model.py             # สคริปต์สำหรับสร้างและฝึกสอนโมเดล Deep Learning (GRU)

├── requirements.txt           # รายการไลบรารีที่จำเป็น
└── README.md                  # เอกสารอธิบายโปรเจกต์
```

   ```bash

   cd fall-detection-system
   ```

## 🚀 การใช้งาน (Usage Instructions)

เพื่อให้ระบบทำงานได้อย่างสมบูรณ์แบบ กรุณาใช้งานตามลำดับขั้นตอน (Pipeline) ดังต่อไปนี้:

### โหมดที่ 1: การรวบรวมและสร้างชุดข้อมูล (Data Collection)
หากคุณต้องการเพิ่มความแม่นยำให้โมเดล หรือต้องการสอนให้โมเดลรู้จักสภาวะการเสียการทรงตัวในมุมมองใหม่ๆ ของพื้นที่ห้องคุณ สามารถเก็บข้อมูลเพิ่มได้โดย:
1. รันคำสั่ง:
   ```bash
   python collect_data.py
   ```
2. ระบบจะเปิดกล้องขึ้นมา ให้คุณทำท่าทางแบบปกติติดต่อกัน (เช่น ยืน เดิน นั่ง) ระบบจะบันทึกสถานะเป็น `0` (Normal)
3. จากนั้นให้จำลองท่าทางการเอนตัวล้ม เอียงตัว หรือสภาวะเสียการทรงตัว ระบบจะบันทึกสถานะเป็น `1` (Loss of Balance)
4. ข้อมูลมุมของข้อต่อในแต่ละเฟรมจะถูกบันทึกและต่อท้ายลงในไฟล์ `fall_dataset.csv` โดยอัตโนมัติ

### โหมดที่ 2: การฝึกสอนโมเดลปัญญาประดิษฐ์ (Model Training)
หลังจากเก็บข้อมูลใน `fall_dataset.csv` จนเพียงพอแล้ว จะต้องนำข้อมูลมาสอนให้ AI (GRU) เข้าใจรูปแบบการล้ม:
1. รันคำสั่ง:
   ```bash
   python train_model.py
   ```
2. สคริปต์จะทำการโหลดข้อมูลจาก CSV แบ่งข้อมูลเป็น Train/Test Set และเริ่มกระบวนการฝึกสอนโมเดล
3. เมื่อเสร็จสิ้น ระบบจะสรุปค่าความแม่นยำ (Accuracy, F1-Score) และทำการเซฟโมเดลออกมาเป็นไฟล์ `fall_model.keras` เพื่อเตรียมนำไปใช้งานจริง

### โหมดที่ 3: ระบบตรวจจับและแจ้งเตือนแบบเรียลไทม์ (Real-time Detection)
เมื่อโมเดลพร้อมแล้ว สามารถนำระบบไปใช้งานจริงเพื่อเฝ้าระวังผู้สูงอายุหรือผู้ใช้งาน:
1. รันคำสั่ง:
   ```bash
   python main.py
   ```
2. ระบบจะเปิดกล้องขึ้นมา พร้อมกับวาดโครงร่างเส้นโครงกระดูก (Skeleton) ทับลงบนตัวบุคคล
3. หากร่างกายอยู่ในสภาวะสมดุล แถบด้านบนจะแสดงสีเขียวพร้อมข้อความ **"NORMAL"**
4. หากโมเดลคำนวณองศาพบว่ามีความสัมพันธ์ของข้อต่อที่ผิดปกติและเสี่ยงต่อการล้ม ระบบจะกระตุ้นการแจ้งเตือนเปลี่ยนเป็นสีแดง และแสดงข้อความ **"WARNING: LOSS OF BALANCE!"** ทันที
5. กดปุ่ม `q` บนคีย์บอร์ดเพื่อออกจากระบบ
